# CS-340_Client_Server_Development
SNHU CS 340 Client/Server Development

How do you write programs that are maintainable, readable, and adaptable? Especially consider your work on the CRUD Python module from Project One, which you used to connect the dashboard widgets to the database in Project Two. What were the advantages of working in this way? How else could you use this CRUD Python module in the future?

In order to write programs that are maintainable, I added comments and docstrings so when I went to make edits later I could quickly understand which sections were resposible for what. I also tried to make the variable names simple and obvious. To make the program readable, I made sure to break it into easy to understand methods like read, delete, etc. I made the program adaptable by making sure I could use it in Project Two as well as One. Some advantages to working this way are that because the database code was written in a separate module, it was faster to test and isolate issues. I could use this CRUD module in the future for another Python application that needs to interact with an animal shelter dataset. 

How do you approach a problem as a computer scientist? Consider how you approached the database or dashboard requirements that Grazioso Salvare requested. How did your approach to this project differ from previous assignments in other courses? What techniques or strategies would you use in the future to create databases to meet other client requests?

I approached the dashboard requirements that Grazioso Salvare requested by first identifying the most important big parts: the database connection, CRUD module, and the interactive widgets. Then I worked them slowly, one at a time, and made sure the major functions worked first, like the pie chart, before I tried to tweak any details, such as the logo size. I approached this project differently because it is larger than others I have done in the past, so I had to break it into smaller more bite sized pieces and test each part separately before putting it all together. In the future I would start by making sure I clearly understand the user requirements before beginning, and I would use a CRUD module again.
